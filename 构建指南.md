## 方法一：GitHub Actions 自动构建（推荐 ✅）

### 第一步：创建 GitHub 仓库

1. 打开 [github.com](https://github.com)，登录你的账号
2. 点击右上角 `+` → `New repository`
3. 仓库名填写：`QuitSmokeApp`
4. 选择 `Public`（免费账户必须用 Public 才能用 GitHub Actions）
5. 不要勾选「Initialize this repository with a README」
6. 点击 `Create repository`

### 第二步：上传代码

在 GitHub 仓库页面，找到「**uploading an existing file**」链接，把以下文件/文件夹拖进去：

```
QuitSmokeApp/
├── app/
│   ├── build.gradle
│   ├── proguard-rules.pro
│   └── src/main/
│       ├── AndroidManifest.xml
│       ├── assets/www/index.html
│       ├── java/com/quitsmoke/app/MainActivity.java
│       └── res/
│           ├── mipmap-hdpi/ic_launcher.png
│           └── values/strings.xml
├── build.gradle
├── settings.gradle
├── gradle/wrapper/gradle-wrapper.properties
├── gradlew
└── .github/workflows/build-apk.yml
```

> 💡 提示：也可以先下载本项目的 zip 包，解压后通过 GitHub Desktop 上传，更方便。

### 第三步：触发构建

代码推送到 `main` 或 `master` 分支后，GitHub Actions 会自动开始构建。

也可以手动触发：
1. 进入仓库页面
2. 点击 `Actions` 选项卡
3. 选择 `Build Android APK`
4. 点击 `Run workflow` → 选择 `main` 分支 → 点击 `Run workflow`

### 第四步：下载 APK

构建完成后（约 5-10 分钟）：
1. 进入 `Actions` 选项卡
2. 点击最新的构建记录
3. 在页面底部找到 `Artifacts` 区域
4. 点击 `QuitSmoke-Debug-APK` 下载
5. 解压后得到 `app-debug.apk`，传到手机安装即可

> ⚠️ Debug 版 APK 需要在手机设置中允许「安装未知来源应用」

## 方法二：直接使用 HTML 文件（无需构建）

`index.html` 可以直接在手机上使用：

1. 把 `index.html` 传到手机（微信文件传输助手、QQ 等）
2. 用手机浏览器（Chrome / Edge）打开
3. 点击浏览器菜单 →「添加到主屏幕」
4. 像 App 一样使用，所有功能正常

---

## 方法三：Expo EAS Build（如果项目改用 React Native）

如果你后续想把 App 做成原生体验（推送通知、健康数据等），可以用 Expo EAS Build：

```bash
# 在终端执行
npm install -g eas-cli
eas login
eas build:configure
eas build --platform android --profile preview
```

构建完成后会生成一个下载链接，直接下载 APK。

## 常见问题

**Q：GitHub Actions 免费吗？**
A：免费账户每月有 2000 分钟构建时间，构建一次安卓 APK 约 5-10 分钟，完全够用。

**Q：APK 安装后提示「恶意软件」？**
A：Debug 版 APK 没有签名，部分手机会提示。这是正常的，点击「继续安装」即可。如需正式签名版，需要在 GitHub Actions 中配置签名密钥。

**Q：如何生成签名版 APK（上架应用商店用）？**
A：需要在 GitHub Actions 中添加签名配置，我可以帮你配置，需要告诉我。

## 项目结构说明

```
QuitSmokeApp/
├── app/                          # 安卓应用模块
│   ├── build.gradle              # 应用构建配置
│   ├── proguard-rules.pro       # 代码混淆规则
│   └── src/main/
│       ├── AndroidManifest.xml   # 应用清单（权限、Activity 声明）
│       ├── assets/www/
│       │   └── index.html       # ⭐ 核心文件：App 所有页面和功能
│       ├── java/com/quitsmoke/app/
│       │   └── MainActivity.java # 安卓原生入口（WebView 加载 HTML）
│       └── res/
│           ├── mipmap-hdpi/     # 应用图标
│           └── values/strings.xml # 应用名称等字符串
├── build.gradle                  # 项目级构建配置
├── settings.gradle               # 项目设置
├── gradle/wrapper/
│   └── gradle-wrapper.properties # Gradle 版本配置
├── gradlew                       # Gradle 执行脚本（Linux/Mac）
└── .github/workflows/
    └── build-apk.yml            # ⭐ GitHub Actions 自动构建配置
```
